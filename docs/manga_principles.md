# Manga Principles

漫画制作エージェントが常に参照する判断原則。

これらは実践用の判断基準であり、全作品に同じ強さで適用する絶対ルールではない。作品の狙い、媒体、読者、作者の強みに合わせて重みづけする。

## 1. 読みやすさは「一致」と「分割」で作る

漫画では、読者がまず吹き出しや大きな顔を見て、その後に絵の情報を拾う。だから、セリフ、表情、情景、オノマトペ、吹き出しの形が同じ感情や事実を指している必要がある。

悪い状態:

- 驚くセリフなのに表情が平静
- 緊迫した状況なのに吹き出しが普通
- 背景で何が起きているかわからないのに説明だけが進む
- 1コマに状況説明、感情変化、伏線、ギャグを全部入れる

よい直し方:

- 1コマで読ませる主情報を1つにする
- 情報が多い場合はコマを割る
- セリフなしのコマや背景コマで読者の負担を下げる
- アップで感情、ミドルで手と姿勢、ロングで位置関係を見せる

## 2. キャラクターは単体ではなく関係性で立つ

主人公を魅力的に見せるには、主人公本人の設定だけでは足りない。周囲のキャラが主人公に何を感じているか、主人公が誰にどう振る舞うかで、読者は主人公を判断する。

確認すること:

- 主人公にしかできない行動があるか
- 主人公の価値観が行動として出ているか
- 他キャラが主人公へ向ける感情が明確か
- 主人公を応援したくなる被害者、弱者、困りごとがあるか
- キャラ同士が自然に会話し、対立し、協力する構造か

## 3. 主人公は好かれる必要がある

主人公は個性的でよいが、読者に嫌われたまま放置してはいけない。嫌な行動、傲慢さ、暴力性、不快なギャグがあるなら、早い段階でケアする。

好かれる主人公の作り方:

- 誰かを助ける
- 助け方がその主人公ならではである
- 強さ、かっこよさ、面白さの「どう」が具体的に見える
- 共感できる弱さと、憧れられる強さが同居している
- 読者が「この人がクラスにいたらいい」と感じられる

## 4. 読み切りは変化量で満足感が決まる

読み切りで重要なのは、1ページ目と最終ページの差である。変化は物理的問題だけでなく、感情、関係性、評価、世界の見え方にも起こせる。

変化の例:

- 被害者が救われる
- 主人公への評価が変わる
- 敵への見方が変わる
- 関係性が敵対から信頼へ変わる
- 問題がマイナスからプラスへ転じる

見せ場を先に決め、その見せ場が一番映える状況を逆算する。キャラ描写を削らないと入らない設定や展開は、構成側を見直す。

## 5. 面白さはメリットからコストを引いたもの

読者は時間と労力を支払って漫画を読む。だから、難しい設定や読みにくいネームは、それ自体がコストになる。

読者コストを下げる方法:

- 大筋がすぐ理解できる
- どこに注目すればよいか明確
- 文字数を削る
- トリッキーな絵を使いすぎない
- 不快要素を入れるなら早くケアする
- ギャグ、オシャレ、設定説明が滑っているなら削る
- 1つの行動にキャラ立て、世界観説明、関係性変化を兼ねさせる

特に新人や読み切りでは、読者との信頼関係がまだ成立していない。長い設定説明や難解な世界観を読んでもらうには、先に興味を作る必要がある。読者の興味が十分に高ければ説明は重くなりにくいが、興味がない状態では説明そのものが離脱理由になる。

## 6. 連載は主人公が動き続ける構造で作る

連載では、毎回アイデアが出る構造が必要になる。主人公が自分から動く目的を持ち、人とぶつかる構造があるほど、話を増やしやすい。

確認すること:

- 主人公が心の底から望んでいる目的は何か
- その目的は毎話の行動を生むか
- 対人の衝突が自然に起きるか
- 新しい敵、依頼、被害者、ライバル、協力者を増やせるか
- 1話完結の変化と長期目標が両立しているか

## 7. 企画はTPOで見る

よい企画は、時代、掲載場所、作者の強みが重なる。

- Time: 今の読者に合う主人公像、題材、空気感か
- Place: 掲載媒体、読者層、競合作品に合うか
- Own: 作者や制作体制の強みが出るか、弱みを隠せるか

AI制作の場合は、AIの強みと弱みもOwnに含める。

AIの強み:

- ビジュアル案や衣装案を大量に試せる
- 舞台や小道具のバリエーションを出しやすい
- プロットやセリフの別案を高速に比較できる

AIの弱み:

- キャラの一貫性が崩れやすい
- 手、視線、表情、距離感の演技が曖昧になりやすい
- コマ間の因果、情報順序、吹き出し導線を自動では管理しにくい

## 8. 作り方は作者に合うフォームを探す

編集者や作家によって、漫画の作り方は違う。企画から考える人も、キャラから考える人も、見せ場から考える人も、題材の掛け合わせから考える人もいる。

エージェントは、1つの型に固定するのではなく、作者が描きやすく、かつ読者に届きやすいフォームを探す。

確認すること:

- 作者が何を描くときに強いか
- 作者の弱みを企画や構成で隠せるか
- 作りたい漫画と向いている漫画がズレていないか
- 企画、キャラ、見せ場、題材のどこから考えると動きやすいか
- 特定作品を例にしなくても、機能として説明できるか

## 9. キャラクターは話を生むエンジンになる

キャラクターができていると、状況や他キャラをぶつけるだけで、そのキャラ固有の反応が生まれる。週刊連載のように短い時間で一定以上のネームを作る場合、これは特に重要になる。

確認すること:

- 主人公がピンチに置かれたとき、普通ではないが納得できる反応をするか
- 他キャラと喧嘩、協力、誤解、競争をさせたとき、会話が自然に動くか
- キャラの「好き」「嫌い」「怖い」「許せない」が行動に出るか
- 事件そのものではなく、事件への反応でキャラ性が見えるか

## 10. ジャンルごとの読者期待を見る

ギャグ・コメディでは、面白さ以前にキャラクターの好感度が重要になる。読者がキャラを好きでなければ、変な行動は笑いではなく不快感になる。

バトル・ファンタジーでは、敵と主人公の価値観の対立を作ると話が動きやすい。敵の目的は、完全に分かりやすいと小さく見え、分からなすぎると読者が付いてこない。読者が追えるフックと、底が見えない怖さの両方を設計する。

設定説明は、キャラクターの生活、行動、選択に絡める。衣食住、仕事、道具、信仰、文化などを1つの行動で見せられると、世界観説明とキャラ立てを兼ねられる。
