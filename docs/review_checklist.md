# Review Checklist

## 企画レビュー

- 主人公は誰か
- 主人公が心の底から望む目的は何か
- 主人公にしかできない行動は何か
- 読者が主人公を好きになる最初の場面はどこか
- 1ページ目と最終ページで何が変化するか
- 見せ場は何か
- 見せ場が一番映える敵、舞台、被害者、状況になっているか
- 対人の衝突が自然に起きるか
- 連載化した場合に話を増やせる構造か
- Time / Place / Own が重なっているか
- 作者やAI制作体制の強みが出る作り方になっているか
- 企画先行、キャラ先行、見せ場先行、題材の掛け合わせのうち、どのフォームが合いそうか
- Time / Place / Own のどれかだけに寄りすぎていないか
- 読者に届く前に作者の描きたいものだけで閉じていないか

## ネームレビュー

- 吹き出しの位置は自然な順番で読めるか
- 吹き出しのツノは誰の発話か迷わないか
- 1コマ内の主情報は1つか
- 絵とセリフが同じ情報を指しているか
- セリフなしのコマでテンポを調整できているか
- ロングで位置関係を見せているか
- ミドルで手、姿勢、動きを見せているか
- アップで表情を見せているか
- 読者が今どこにいて、誰が誰に向かって話しているか迷わないか
- 文字数を削れる吹き出しはないか

## キャラクターレビュー

- 主人公の共感ポイントは何か
- 主人公の憧れポイントは何か
- 主人公を推せるポイントは何か
- 主人公を愛でたくなるポイントは何か
- 主人公の嫌われそうな点はケアされているか
- 主人公の価値観は行動で見えるか
- 何に怒り、何をされたら喜ぶかが見えるか
- 周囲のキャラは主人公にどんな感情を向けているか
- キャラ数は足りているか、多すぎないか
- 独り言や説明セリフを減らせる関係性になっているか
- 事件へのリアクションでキャラ性が出ているか

## 読み切りレビュー

- 冒頭に大きい絵、面白い絵、顔の大きいコマがあるか
- 読み終わったあと、主人公を好きになっているか
- 変化は物理、感情、関係性、評価の複数軸で起きているか
- 被害者や救われる対象が記号で終わっていないか
- 見せ場に向けた逆算ができているか
- ミスリードは十分に仕込まれているか
- 予想を超えるポイントがキャラ立てにつながっているか

## バトルものレビュー

- 主人公、敵、被害者の三角形があるか
- 被害者は読者が救いたくなる存在か
- 敵の強さや怖さは絵で伝わるか
- 敵の価値観は主人公と対立しているか
- 敵の目的は読者が追えるフックと、底が見えない怖さの両方があるか
- 主人公の勝ち方がその主人公らしいか
- 戦闘前後でマイナスからプラスに転じるものがあるか
- 読者の予想を超えるポイントがあるか
- その意外性は後出しではなく、前から仕込まれているか
- 技名、残虐描写、死亡シーンは作品の温度感に合っているか

## ギャグ・コメディレビュー

- キャラの好感度がギャグより先に作られているか
- 変な行動が、不快ではなく笑いとして読める状態になっているか
- 1話完結の入りやすさを活かせているか
- 毎回のギャグでキャラの新しい面が見えるか
- ツッコミ役、観察役、被害者役が機能しているか

## 読者コストレビュー

- 大筋をすぐ理解できるか
- 各シーンで注目点が明確か
- 読者の時間と労力に見合うメリットがあるか
- 設定説明より先に興味が作れているか
- 1つの行動がキャラ立て、世界観説明、関係性変化を兼ねているか
- 不快要素がある場合、早めにケアされているか

## AI生成レビュー

- 各コマの主情報がプロンプトに明記されているか
- キャラの表情、視線、手の演技、姿勢が指定されているか
- コマ間で衣装、髪型、小物、傷、位置関係が維持されているか
- セリフに頼りすぎず、絵で伝える情報が分担されているか
- 生成絵の見栄えと、漫画としての読みやすさを別々に評価しているか
- AIが生成しにくい手の演技、視線、距離感、吹き出し導線を人間側の指示で補えているか
