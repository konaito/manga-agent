# 吹き出し・文字容器の描き分け調査メモ

漫画の「中身の文体」ではなく、**絵の上でセリフ／心の声／地の文をどう描き分けるか**の調査。AI画像生成では `kind` をデータ上で分けても、プロンプトに形状指示がなければ全部「白い箱＋文字」に潰れる。

## 調査から得た原則

### 1. 大分類は3系統

| 系統 | 用途 | 読者への意味 |
|---|---|---|
| 声に出す系 | セリフ | 他者にも聞こえる |
| 心理描写系 | 心の声・モノローグ | 声に出していない |
| ナレーション系 | 地の文・状況説明 | 語り手／客観情報 |

形を変えるのは装飾ではなく、**声の質を視覚的に伝える演出**である。

### 2. 作品内で形状セットを固定する

AnimeOutline も「1作品で1セットのスタイルを選び、通篇統一する」と明記している。  
商業誌でも、通常セリフ用フォント・枠線太さ・地の文枠は作品ごとに固定し、**例外形状は感情の強調時だけ**使う（egaco / 商業誌フォント統一の解説）。

AI生成でも同じ。ページごとに oval / 四角 / 雲形が混在すると読者は混乱する。

### 3. 描き分けの定番（本作のデフォルト）

| kind | 形状 | しっぽ | 配置 | 禁止 |
|---|---|---|---|---|
| `dialogue`（speaker あり） | 楕円吹き出し | 尖ったツノ → 口元 | 話者の頭の近く | 四角枠、雲形 |
| `monologue` | 雲形（心の声） | 小さな泡3〜5個 → 頭 | 思考キャラの頭の近く | 口への尖ツノ、地の文枠 |
| `caption` | 斜め四角（地の文） | なし | コマ端・余白 | キャラ接続、雲形 |
| `sfx` | 描き文字 | — | 効果と一体化 | 白い吹き出し枠 |
| `ui` | 画面内UI | — | デバイス画面上 | 吹き出し化 |

### 4. 思考と結果（地の文）を混同しない

`monologue` と `caption` は形状だけでなく、**情報の種類**が違う。

| 分類 | 中身 | kind | 描き方 |
|---|---|---|---|
| 思考 | 今この瞬間の内心・迷い・決意・自己欺瞞 | `monologue` | 雲形＋泡しっぽ→頭 |
| 結果・語り | 時間・場所・行動の結果・事実の叙述・回想の締め | `caption` | 斜め四角・コマ端・非接続 |

判断基準（最重要）:

- **caption（地の文）**: 語り手の俯瞰。過去形・状態・結果・事実・「〜ていた」「〜した」「〜がある」。**一人称でも地の文なら caption**。
- **monologue（雲形）**: 同一コマの「今」だけ頭の中で未整理の短い断片（例:「……まずい」）。**本作では原則使わない**。

「客なのか、決めかねていた」は**今迷っているのではなく、後から状態を述べている**ので caption。雲形にしない。

具体例（すべて caption）:

- 「何も考えていなかった」
- 「テレビをつけなかった」
- 「客なのか、決めかねていた」
- 「放っておく選択肢が遠のいた」
- 「彼の感謝が、私の言葉を正しくした」
- 「見ないことを、選んだ」
- 「知らない若者の荷物がある」
- 「この夜から、私の日常は侵食されていった」

### 5. モノローグと地の文を形状で混同しない

よくある失敗:

- モノローグを「ツノなし四角」で描く → **地の文と同型**になり判別不能
- 地の文をキャラの頭の近くに置く → **心の声に見える**
- セリフと同じ楕円にツノだけ変える → 差が小さすぎる

区別の鍵は **形状＋接続＋配置** の3点セット。

- **心の声**: 雲形 ＋ 泡しっぽ ＋ キャラの頭付近
- **地の文**: 斜め四角 ＋ 接続なし ＋ コマ端

### 6. 統一感のルール

作品を通して固定するもの:

- 枠線: 黒・実線・同じ太さ
- 塗り: 白
- 字体: ゴシック系・縦書き（本作）
- 内側余白: 文字と枠の間隔を一定

変えてよいもの（kind ごと）:

- 外形（楕円 / 雲 / 斜め四角）
- しっぽの有無・形状
- 配置ゾーン（頭付近 vs コマ端）

### 7. 配置と読み順

- 日本漫画: 吹き出しも右上 → 左下の視線流れに沿う
- ツノは必ず話者の口元へ（セリフのみ）
- 複数キャラのセリフ: 高さをずらし、ツノ交差を避ける
- 地の文は絵の主役（顔・手）を隠さないコマ端へ

### 8. 感情別の例外形状（第1話では控えめに）

| 感情 | 形状 | 第1話での使用 |
|---|---|---|
| 叫び | ギザギザ | 使わない（静かな会話劇） |
| 小声 | 点線枠 | 必要なら「すいません」系 |
| 動揺 | 波線枠 | 原則使わない |
| 強調モノローグ | フラッシュ枠 | P10/P16 の核心1箇所のみ検討可 |

第1話「夜の呼び水」は会話劇なので、**基本3形状（楕円・雲・斜め四角）の統一を優先**し、感情枠は増やさない。

## 本作（鬼畜 ep01）への適用

| 場面 | 正しい容器 |
|---|---|
| 「すいません」「うち、来る？」 | 楕円＋口ツノ |
| 「二年前の十月。三十歳。」「深夜一時」「翌朝」 | 斜め四角・コマ端 |
| 「ピロン」 | 描き文字 |
| 「何も考えていなかった」「テレビをつけなかった」「客なのか、決めかねていた」「放っておく選択肢が遠のいた」「彼の感謝が、私の言葉を正しくした」「見ないことを、選んだ」「知らない若者の荷物がある」「二年続くとは思っていなかった」「この夜から、私の日常は侵食されていった」 | 斜め四角・コマ端（結果・語り・俯瞰） |

**本作 ep01 では monologue（雲形）を使わない。** 語りも結果もすべて caption。

## AI生成パイプラインへの落とし込み

1. `storyboard.json` の `kind` / `speaker` を維持
2. `build_prompt()` が kind ごとに **形状・しっぽ・配置** を英語で明示
3. `pro_panel_craft.md` に作品固有の統一ルールを書く
4. QA で「セリフとモノローグが同型」「地の文がキャラ頭付近」などを見る

## チェックリスト

- [ ] 地の文はすべて斜め四角＋コマ端＋非接続か
- [ ] 1ページ内で枠線太さ・字体・余白が統一されているか
