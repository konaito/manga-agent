# 漫画脚本レビュー・チェックリスト

`script.md` 完成後、**ネームに進む前に必ず**このチェックリストでレビューする。

成果物: `production/spec/script_review.md`（各話）

## 判定

- **ship**: ネームへ進める
- **revise**: 脚本を直してから再レビュー
- **blocked**: story bible / 翻案設計へ戻る

## チェック項目

### 会話劇

- [ ] 重要場面が「説明」ではなく「応酬」になっている
- [ ] 1つの決断が1発話で終わっていない（提案→拒否→押し返し→沈黙→受諾）
- [ ] 各セリフに相手の `reaction_beats` がある
- [ ] `must_say` のセリフが会話の流れの中に自然に組み込まれている

### 地の文 vs 思考（kind 設計）

- [ ] 非セリフ行が `caption`（結果・語り・俯瞰）か `monologue`（同一コマの未整理断片）か、脚本段階で分類されている
- [ ] 過去形・「〜ていた」「〜した」「〜がある」はすべて **caption 候補**として扱っている
- [ ] 「客なのか、決めかねていた」のような**後から述べる状態**を monologue にしていない
- [ ] 本作方針: **原則 monologue（雲形）は使わない**。迷いも俯瞰も caption

### 漫画化

- [ ] 各 `scene_block` に入口・出口がある
- [ ] `page_intent` にページごとのヒキがある
- [ ] 絵だけで見せる情報をセリフで説明していない
- [ ] `must_not_say` に反するセリフがない

### 連載

- [ ] この話の `emotional_delta` が adaptation_design と一致
- [ ] 話末ヒキが次話への問いを残す
- [ ] 新キャラ登場が1人以内（ep2以降）

## レビュー記録フォーマット

```md
# 第X話 脚本レビュー

- 判定: ship | revise | blocked
- レビュー日:
- 入力: script.md

## 良い点

## 修正必須

## 修正推奨

## ネームへ渡す注意
```
